Sitios como facebook, ig  o netflix funcionan muy rapido, esto se debe a que el front esta optimizado con REACT



REACT

es una biblioteca escrita en JS

para facilitar la creacion de componentes interactivos  reutilziables en el front de tu aplicacion

Se puede usar del lado del cliente y tambien del servidor


Esta construido en torno a hacer funciones, las cuales van a tomar actualizaciones del estado de la aplicacion

y van a traducir estos cambios en una representacion vitual de la misma


Siempre que react es informado de un cambio, va a ejecutar todas las funciones necesarias para realizar una nueva

representacion virtual del documento 


Ese cambio se ve reflejado directamente en el DOM, la pagina muestra info actualizada 


La libreria trabaja con un concepto llamado VIRTUAL DOM, esti lo que hace es hacer una copia exacta del DOM y hacer

una comparacion del VIRTUAL DOM y del DOM actual y cuando se modifica el DOM real, se modifica el VIRTUAL DOM


Lo hace de manera automatica y optimiza mucho el rendimiento ya que solo actualiza ese componente.


** DIFFING **

 El proceso de comparar el virtual DOM y el DOM actual


** RECONCILIATION **

 El proceso de actualizacion 






La instalacion se puede hacer directo desde la terminal ya que se encuentra en npm


Te ubicas en el directiorio donde deseas crear el proyecto y realizas el siguiente comando:

- npm creat-react-app "nombre-proyecto"

o sino

- npm init react-app "nombre-proyecto"


Es uno o el otro, no ambos, se recomienda escribir todo en minusculas separados por guiones, demora en descargarse



Una vez instalado todo, la carpeta tiene una estructura similar a como veniamos trabajando


En las diferentes carpetas vamos a encontrar archivos importantes:


** PUBLIC **

 - index.html
            tiene un html tradicional que tiene un div con id="root" que es muy importante ya que alli se renderiza

            toda la aplicacion, No se toca ni modifica


** SRC **

 -index.js
            alli vamos a encontrar varias lineas con la sentencia import, es como el require de NODE, de la cual se 

            trae algo desde otra libreria y se lo almacena para poder utilizarlo

            Tambien tiene el metodo .render del objeto ReactDom allí se  indica que se utilizara el element con 
            
            ID "root"

            ReactDom.render(<App />, document.getElementById('root'))

            
            App es un componente y se esta importando al archivo index.js en esa linea


 -app.js
            hay una estructura similar a la de index.js pero tiene una funcion app que se importa al final del archivo

            y dentro de la funcion se retorna una estructura "html" es una interpretacion del lenguaje html



Con npm start vamos a ejecutar la aplicacion y la vamos a poder ver desde un navegador, este toma los cambios

automaticamente
